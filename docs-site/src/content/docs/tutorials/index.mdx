Start here after install. The tutorial path is intentionally small: run one visual graph, compare expected output, learn the few concepts needed to scale it, then move to debugging, Hub reuse, or Golden robot examples.

## What You Should Have After Each Stage

| Stage | Artifact | You are ready to continue when... |
| --- | --- | --- |
| Visual quickstart | stdout detection events, optional Rerun viewer | The mock command is deterministic and the live command either opens Rerun or falls back to stdout. |
| Examples and results | expected outputs for Flow, stepping, perception, graph render, replay | You can tell whether a command succeeded without reading source first. |
| Flow fundamentals | a typed Python mental model | You know where user logic lives: inside `Flow.step(...)`. |
| Time and sync | clocks and edge sampling vocabulary | You can explain which Flow wakes itself and how upstream history is sampled. |
| Debug and visualize | `artifacts/tutorial_perception.html`, step traces, replay logs | You can inspect the graph before blaming backend scheduling. |
| Hub and reuse | manifest/registry boundary | You know what is reusable as a pack and what is still source-only. |

## AI-Agent Route

Agents should avoid broad repo spelunking on the first pass. Use this deterministic route instead:

```bash
pixi run demo-webcam-detection-mock
pixi run docs-tutorial-perception-html
pixi run demo-stepper
```

Then read [`llms.txt`](/llms.txt), inspect the rendered HTML graph, and only move to live webcam, Rerun, simulator, or Golden examples after the mock path works.
