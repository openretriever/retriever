## Quick Start

Run the source-checkout demo first. It proves the runtime graph without camera permission, GUI windows, Rerun, or robot hardware.

```bash
git clone https://github.com/openretriever/retriever.git
cd retriever
pixi install
pixi run demo-webcam-detection-mock
```

<details class="explain-details">
  <summary>What does this do?</summary>
  <div>
    <p><code>pixi install</code> creates the reproducible repository environment. <code>pixi run demo-webcam-detection-mock</code> runs a small graph: synthetic camera frames feed a color detector, then detections are printed to stdout. Use the webcam/Rerun path only after this smoke works.</p>
  </div>
</details>

After `retriever-core==0.0.1` resolves from PyPI, the library install path becomes:

```bash
python -m pip install retriever-core
```

## Why Retriever For Robotics

<div class="check-list">
  <p><strong>No single global loop.</strong> Each Flow can run on its own clock, so cameras, policies, planners, monitors, and controllers do not need to pretend they share one timestep.</p>
  <p><strong>Explicit data handoff.</strong> Each edge declares how upstream event history becomes the one input consumed by <code>step(...)</code>.</p>
  <p><strong>Debuggable before deployment.</strong> Render the graph, step locally, use stdout/Rerun, record inputs, and replay the same run.</p>
  <p><strong>Ordinary Python logic.</strong> User code stays in typed classes with synchronous <code>step(...)</code>; runtime timing lives around the graph.</p>
</div>

## How Retriever Compares

| Pattern | Good at | Retriever adds |
| --- | --- | --- |
| ROS callbacks | Device integration and message transport. | Typed graph boundaries, explicit sampling, local stepping, and replayable handoff. |
| One Python loop | Simple demos and scripts. | Multi-rate modules without a fake global timestep. |
| Actor/DAG systems | Distributed asynchronous execution. | Closed-loop cyclic graphs with clocks, sync policies, and robot-friendly debugging. |
| Notebook prototypes | Fast iteration. | A path from cells to inspected, recorded, reusable runtime graphs. |

## The Four Objects

<div class="card-grid">
  <a class="info-card" href="/concepts/flow/"><strong>Flow</strong><span>A typed Python class with local state and a synchronous <code>step(...)</code> method.</span></a>
  <a class="info-card" href="/concepts/time-and-sync/"><strong>Clock</strong><span>Each Flow declares when it runs; there is no global robot timestep.</span></a>
  <a class="info-card" href="/concepts/time-and-sync/"><strong>Sync policy</strong><span>Each edge declares how upstream event history becomes one aligned input.</span></a>
  <a class="info-card" href="/concepts/runtime/"><strong>Pipeline</strong><span>The graph that validates, visualizes, steps, replays, and runs.</span></a>
</div>
